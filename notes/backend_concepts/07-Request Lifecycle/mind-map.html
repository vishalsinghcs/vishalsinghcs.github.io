Request Lifecycle Mind Map

A visual cheat sheet of controllers, services, repositories, middlewares, and request context.

flowchart LR Root((Request
Lifecycle)) %% Controller / Handler Root --> Ctrl{Controller / Handler} Ctrl --> C1[Bind & Deserialize
JSON → native struct] Ctrl --> C2[Validate & Transform
Check all incoming data] Ctrl --> C3[Call Service Layer
Delegate processing] Ctrl --> C4[/Send HTTP Response
200, 400, 500/] %% Service Layer Root --> Svc{Service Layer} Svc --> S1([Business Logic
Emails, calculations, rules]) Svc --> S2([Orchestration
Calls multiple repositories]) Svc --> S3([No HTTP code
Pure functions only]) %% Repository Layer Root --> Repo{Repository Layer} Repo --> R1{{Single Responsibility
One method, one DB operation}} Repo --> R2{{Constructs DB queries
No business logic here}} %% Middleware Root --> MW{Middlewares} MW --> MW1[CORS
Block unauthorized origins] MW --> MW2[Auth
Verify token → 401 or next] MW --> MW3[Rate Limit
Exceed threshold → 429] MW --> MW4[Logging
Log each request early] MW --> MW5[/Error Handler
Must be last in chain/] %% Request Context Root --> Ctx{Request Context} Ctx --> X1([Request-scoped store
Key-value, isolated per request]) Ctx --> X2([Set by middleware
e.g., userId after auth]) Ctx --> X3([Read by handlers
