Interview Questions

05 - Authentication & Authorization: Security, Identity, and Access Control

These questions test your ability to design secure systems, debug complex distributed identity issues, and evaluate architectural trade-offs in modern authentication flows.

1. Scenario-based Questions (40%)

  • Your frontend sends a JWT in the Authorization header, but the backend rejects it as "expired". The user just logged in 5 minutes ago, and the token expiration is set to 1 hour. What could be the issue?
  • Users report that when they log in to your web application from a new device, they are sometimes randomly logged out from their old device. How would you investigate this session handling behavior?
  • You are building an e-commerce site where users can add items to a cart as an anonymous "guest" before creating an account. How do you handle authentication and session merging when they finally click "Sign Up"?
  • An attacker steals a user's JWT from local storage via an XSS vulnerability. How can you detect or mitigate the impact since stateless JWTs cannot easily be revoked?
  • Your distributed Redis cluster holding stateful session IDs goes down completely. The web application starts failing all authenticated requests. How do you recover or design for this failure?
  • A user complains that they can see the "Admin Dashboard" button in the UI, but when they click it, the API returns a 403 Forbidden error. How do you debug the disconnect between the UI state and the backend RBAC?
  • Your API is rate-limited, but an attacker is spamming the /login endpoint. You notice they are trying to measure response times to guess valid emails (a timing attack). How do you implement a constant-time defense in your existing application?
  • A microservice architecture uses OAuth 2.0. Service A requests a token from the Auth Server and passes it to Service B. Service B trusts the token, but you suspect a "confused deputy" problem. How do you validate the token's audience securely?

2. "What if?" Questions (20%)

  • What would happen to modern microservice architectures if JWTs didn't exist and every service had to validate stateful sessions with a central database?
  • What if HTTP cookies were entirely removed from browsers for privacy reasons? How would we implement automated stateful authentication?
  • What if you designed an RBAC system where permissions were tied directly to users instead of abstract roles? How would it scale as your company grows to 10,000 employees with changing job titles?
  • What if OAuth 2.0 didn't have the concept of "scopes"? What would be the security implications for third-party integrations like "Sign in with Google"?

3. Debugging Questions (15%)

  • A user logs in successfully, but the browser does not attach the session cookie to subsequent API requests. The API and the frontend are hosted on different subdomains. What headers or cookie attributes are missing?
  • You implement a hybrid approach where you store blacklisted JWTs in Redis. However, users who were banned are still able to access the API for up to 5 minutes after the ban. Where is the bug in your invalidation logic?
  • An API endpoint checks if (user.role === 'admin') but the database stores roles as an array [ 'editor', 'admin' ]. A user with the admin role is getting a 403. How did this happen and how would you fix the authorization middleware?

4. Design Questions (15%)

  • Design a secure "Password Reset" flow. Discuss how you generate the reset token, where you store it, how long it lives, and how you prevent abuse of the endpoint.
  • Design an authentication system for a smart TV application where typing a complex password via the remote control is difficult. How do you delegate the login flow to the user's smartphone?
  • Design a system that temporarily grants a third-party vendor access to upload a specific file to your secure S3 bucket without giving them permanent AWS IAM credentials.

5. Architecture & Trade-off Questions (10%)

  • When would you choose to use opaque tokens (random strings validated via a database lookup) instead of structured tokens (JWTs) for your internal microservice authentication?
  • Why might an enterprise company choose to implement their own custom authentication service instead of using a robust managed provider like Auth0 or AWS Cognito?
