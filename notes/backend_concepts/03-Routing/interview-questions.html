Interview Questions

03 - Routing: Dynamic Mapping & API Architecture

These questions test engineering thinking, debugging ability, production experience, and trade-off analysis. Try to answer them thoroughly without looking up definitions.

1. Scenario-based Questions (40%)

  • You notice two API routes exist in your codebase: GET /api/users/:id and GET /api/users/active. However, requests to /active keep hitting the :id handler and throwing "Invalid ID" errors. Why is this happening, and how do you fix it?
  • Your team is building a search endpoint. A junior developer suggests using POST /api/search to send complex search criteria in the request body, while another suggests using GET /api/search?q=.... What are the caching and semantic trade-offs of each approach?
  • You need to deprecate an old endpoint /api/v1/checkout and migrate users to /api/v2/checkout. However, mobile apps cannot be updated instantly. Describe a safe, phased migration strategy using route versioning.
  • Your API server is receiving thousands of requests for GET /api/products/null. The client team insists they aren't explicitly sending the string "null". Where in the routing and frontend layer is this coming from and how do you handle it gracefully?
  • You deploy a new route GET /dashboard/:userId/stats. The next day, you discover users are manually changing the userId in the URL and viewing other users' private stats. How do you secure dynamic route parameters at the handler level?
  • An endpoint GET /api/reports returns a huge dataset and occasionally crashes the server. You decide to implement pagination via query parameters. How do you design the routing contract (URL and response structure) to inform the client about the next page?
  • A third-party webhook pushes data to your server via POST /webhooks/payment, but the URL sometimes includes query parameters like ?source=stripe. Your server rejects it with a 404. How should the route matching be configured to ignore or accept query parameters during path matching?
  • Your team decides to split a monolithic backend into microservices. How do you configure a reverse proxy routing table to route /api/users/* to the User Service and /api/orders/* to the Order Service transparently?

2. "What if?" Questions (20%)

  • What if a backend framework didn't support dynamic path parameters (like :id) and you could only use static strings? How would this impact your URL structure and database querying logic?
  • What if you forgot to include a catch-all wildcard route (/*) at the end of your server configuration? What exact HTTP response would the client receive, and what fingerprinting security risks might it expose?
  • What if REST conventions didn't exist and every action was routed through a single endpoint like POST /api/action with the intent specified in the body? What caching, monitoring, and infrastructure optimizations would you lose?
  • What if query parameters were strictly limited by the browser to a maximum length of 20 characters? How would you redesign complex filtering and sorting API endpoints?

3. Debugging Questions (15%)

  • A client sends a request to GET /api/users/123/posts/456. The server logs show the request arrived, but it returned a 404. You verify that user 123 and post 456 both exist in the database. Where in the routing layer should you start debugging?
  • You have pagination implemented via query parameters (?page=1&limit=50). Suddenly, the database starts crashing due to heavy queries. Upon inspection, you realize a malicious actor sent ?limit=1000000. How do you prevent this at the routing layer before it hits the database?
  • A developer adds a new route GET /api/files/* to serve static images. Suddenly, all other API routes like GET /api/users start returning static file routing errors instead of JSON. What is the execution order bug in the routing setup?

4. Design Questions (15%)

  • Design a URL routing structure for an e-commerce platform that needs to filter products by category, sort them by price, and support pagination. Should these be path parameters or query parameters? Write out the example URLs.
  • Design a RESTful routing schema for a collaborative document editor where users can invite other users to edit specific documents. Detail the endpoints for inviting, removing, and listing collaborators on a document.
  • Design a routing strategy to support A/B testing on your backend. How would you route 10% of users to a new experimental handler without changing the endpoint URL on the frontend?

5. Architecture & Trade-off Questions (10%)

  • When building a public API, what are the trade-offs between placing the version number in the route path (e.g., /v1/users) versus placing it in a custom HTTP header (e.g., Accept-Version: v1)?
  • Why might a company choose to use GraphQL (which typically routes everything through a single /graphql endpoint) over traditional RESTful routing? What network-level routing features do they sacrifice by doing this?
