Inverted Index

Definition

An inverted index is a data structure that maps content (such as words or terms) back to its location in a set of documents, effectively flipping the search problem upside down.

How it Works

Instead of scanning every document to see if a term exists, an inverted index creates a global dictionary of all terms found across all documents during the storage phase.

Step-by-step Flow

  1. Ingestion: When a new document (e.g., a book or product) is added, the system scans its content.
  2. Tokenization: The text is broken down into individual terms (words).
  3. Indexing: For each term, the system records exactly which documents contain it and where it appears.
  4. Retrieval: When a user searches for a term, the system simply looks up the term in the dictionary and instantly retrieves the list of matching documents.

Advantages

  • Blazing Fast: Lookups happen in milliseconds, regardless of how large the total dataset grows.
  • Scalable: It efficiently handles millions of documents.
  • Foundation for Relevance: Since it tracks the frequency and location of terms, it allows for sophisticated ranking algorithms.

Apache Lucene

The inverted index is not a new concept; it has been researched since the 1960s. Today, the most prominent open-source implementation of the inverted index is Apache Lucene. Elasticsearch, along with many other full-text search tools, is built on top of the Lucene engine.

Mental Model

An inverted index is exactly like the index at the back of a textbook. Instead of reading the whole book to find where "Machine Learning" is mentioned, you look it up in the back and it gives you the exact page numbers instantly.

Important Terms

Inverted Index — A dictionary-like data structure mapping terms to document locations.

Apache Lucene — The core text search engine technology that powers Elasticsearch.

Key Takeaways

  • Inverted indexing flips the search problem from "find term in documents" to "look up documents by term."
  • It processes text heavily during insertion (write) to make retrieval (read) incredibly fast.

Quick Revision

  • Creates a map of Word → List of Documents.
  • Powers full-text search engines like Elasticsearch.
  • Reduces search time from seconds to milliseconds.
