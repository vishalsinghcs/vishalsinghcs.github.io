Types of Configuration

Not all configurations are created equal. Understanding the different types helps in determining their storage, security measures, and access patterns.

1. Application Settings

The most common kind of configuration that controls the core behavior of the backend application itself.

  • Log Level: `debug` for development, `info` for production.
  • Port: e.g., running on `8080` locally versus standard ports in production.
  • Connection Pool Size: Maximum number of simultaneous database connections.
  • Timeouts: Maximum time (e.g., 60 seconds) the server waits before dropping a long-running request.

2. Database Configuration

Includes all the details required for the backend to connect to its primary database.

  • Host and Port.
  • Username and Password.
  • Database Name.
  • Query execution timeouts.

3. External Services

Configurations required to integrate with third-party tools and APIs.

  • Email Providers: Resend, Mailchimp API keys.
  • Payment Processors: Stripe secret keys.
  • Authentication Providers: Clerk, Auth0 API keys.

4. Feature Flags

Used to dynamically enable or disable specific features of an application without redeploying code.

  • A/B Testing: Enabling a new checkout flow for 50% of users.
  • Geographic Rollouts: Enabling a feature for users in the US but keeping it disabled in India.

5. Other Configurations

  • Infrastructure Config: DevOps related settings.
  • Security Config: JWT secrets, session secrets, encryption keys.
  • Performance Tuning: Environment variables for maximum CPU usage or memory limits (e.g., Go `GOMAXPROCS`).
  • Business Rules: Centralized rules like "maximum order amount" or "discount limits".

Mental Model

Imagine your application as a restaurant. The Application Settings are the opening hours and seating capacity. Database Config is the lock to the pantry. External Services are your suppliers (meat, veggies). Feature Flags are special menu items you're testing on VIP guests.

Important Terms

Feature Flag: A configuration technique that allows you to enable or disable functionality remotely without changing code.

Connection Pool: A cache of database connections maintained so that they can be reused when future requests to the database are required.

Key Takeaways

  • Categorizing configs helps apply appropriate security.
  • Not all configs are secrets (e.g., feature flags).
  • Application configs change the behavior of the server.
  • Database and external services configs handle connectivity.

Quick Revision

  • App Settings: Timeouts, ports, log levels.
  • Database: URL, username, password, host.
  • External: Stripe, Mailchimp API keys.
  • Feature Flags: A/B testing, selective rollouts.
  • Security: JWT secrets, session keys.
