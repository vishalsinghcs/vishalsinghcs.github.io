Backend vs Frontend

The frontend is the client-side code executed by the user's browser, while the backend is the centralized server-side code that persists data and executes heavy business logic.

Why do we need backends?

To centralize state, securely persist data, and orchestrate actions between multiple users. Without a central authority, users cannot interact securely or reliably (e.g., User A liking User B's post).

How frontends work

  1. Initial Request: Browser fetches the primary HTML document.
  2. Resource Fetching: Browser parses HTML and fetches linked resources (CSS, JS, Fonts).
  3. Painting: CSS is applied to style the UI.
  4. Hydration: JavaScript is executed to add event listeners and logic.
  5. Execution: All application code runs entirely within the client's browser sandbox.

Why can't we write backend logic in the frontend?

Restriction Description
Security & Sandboxing Browsers isolate code from the OS. Frontend code cannot access the file system, environment variables, or sensitive logs. If not isolated, malicious sites could easily steal user files.
CORS Policies Browsers restrict JavaScript from freely calling external APIs across different domains to prevent unauthorized actions and data theft.
Database Connections Native database drivers require binary data handling, socket connections, and persistent connection pooling. Browsers cannot maintain these. Furthermore, millions of clients opening direct connections would crash the database.
Computing Power Frontends run on unpredictable devices (e.g., old smartphones). Heavy business logic would cause lag and crashes. Backends have scalable, dedicated compute resources.

Mental Model

Think of the frontend as a restaurant menu and ordering tablet provided to customers. The backend is the central kitchen that has access to all ingredients (data), manages orders from all tables simultaneously, and securely prepares the food.

Important Terms

  • Connection Pool — A cache of database connections maintained by the backend server so connections can be reused.
  • Hydration — The process of attaching JavaScript event listeners to a static HTML document.
  • Sandbox — An isolated execution environment used by browsers.

Key Takeaways

  • Frontends execute code in the user's browser runtime.
  • Backends execute code in a controlled server runtime.
  • Security, database pooling, and network restrictions dictate that core logic must remain on the backend.

Quick Revision

  • Frontend = Client runtime (Browser).
  • Backend = Server runtime (Centralized).
  • Frontends cannot securely access DBs directly.
