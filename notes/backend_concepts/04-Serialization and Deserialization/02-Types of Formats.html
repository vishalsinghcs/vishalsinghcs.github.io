Types of Serialization Formats

Just as there are many programming languages and databases, there are multiple standards for serialization depending on the use case.

Categorizing Formats

Serialization formats broadly fall into two categories: Text-Based and Binary.

flowchart TD Root[Serialization Standards] Root --> Text[Text-Based Formats] Root --> Binary[Binary Formats] Text --> Json[JSON] Text --> Yaml[YAML] Text --> Xml[XML] Binary --> Proto[Protobuf]

1. Text-Based Formats

These formats are designed to be human-readable. You can easily open them in a text editor, read the contents, and debug them manually. Because they prioritize readability, they are slightly larger in file size and require slightly more compute to parse.

  • JSON (JavaScript Object Notation): The undisputed king of modern web APIs. It is lightweight, widely supported, and native to JavaScript. We will dive deeper into this in the next lesson.
  • XML (eXtensible Markup Language): Older and highly structured (looks like HTML). Still heavily used in legacy enterprise systems, SOAP APIs, and financial software.
  • YAML (YAML Ain't Markup Language): Extremely readable due to its reliance on indentation rather than brackets. Mostly used for configuration files (like Docker or CI/CD pipelines) rather than real-time network requests.

2. Binary Formats

These formats are not human-readable. They compile data directly into dense binary representations. Because they skip the human-readable text step, they are incredibly fast to parse and result in tiny payload sizes.

  • Protobuf (Protocol Buffers): Developed by Google, Protobuf is the primary serialization format used in high-performance microservice communication, particularly alongside frameworks like gRPC.

Choosing the Right Tool

The backend engineering ecosystem is vast. You don't need to master every single technology on day one. When starting out, if you master HTTP Rest APIs communicating via JSON and storing data in a relational database like PostgreSQL, you are perfectly equipped for ~80% of modern backend roles.

You can always adopt Protobuf or YAML as your career demands it.

Mental Model

Text formats are like reading a well-formatted book—anyone can understand it, but it takes up a lot of space. Binary formats are like shorthand code—unintelligible to humans, but incredibly dense and fast for machines to process.

Important Terms

  • JSON: JavaScript Object Notation; the default standard for HTTP REST APIs.
  • Protobuf: Protocol Buffers; a dense binary format built by Google, often used in gRPC microservices.
  • YAML: A human-readable text format heavily relying on indentation, used mostly for configuration.

Key Takeaways

  • JSON is the king of web APIs. Master it first.
  • Text-based formats trade network efficiency for developer debugging ease.
  • Binary formats are used when nanosecond speed and minimum bandwidth are critical.

Quick Revision

  • Text-Based: Human-readable, heavier. (JSON, XML, YAML)
  • Binary: Machine-optimized, extremely fast/small. (Protobuf)
  • JSON is the most popular choice for HTTP REST APIs.
  • YAML is the most popular choice for configuration files.
  • Protobuf is popular for internal microservice communication (gRPC).
