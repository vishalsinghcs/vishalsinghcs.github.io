Types of Background Tasks

Different patterns for executing offline workflows.

1. One-off Tasks

A single task triggered by a specific event in the request-response lifecycle. These are the most common type of tasks.

  • Example: Sending a welcome email after a user signs up.

2. Recurring Tasks (Cron Jobs)

Tasks that execute periodically at specific intervals.

  • Example: Sending a weekly analytics report every Sunday at midnight, or cleaning up abandoned database sessions at the end of the month.

3. Chained Tasks

Tasks that have a parent-child relationship, where a task can only execute once its parent successfully finishes.

  • Example: In a video platform, Task A (Video Encoding) runs first. Once finished, it triggers Task B (Thumbnail Generation) and Task C (Transcription Generation) in parallel.

4. Batch Tasks

A single trigger spawns multiple similar tasks, or one large task acts on a large batch of data concurrently.

  • Example: An "Account Deletion" trigger that spawns dozens of smaller tasks to delete a user's specific assets across different microservices. Another example is sending a monthly newsletter to thousands of users simultaneously.

Mental Model

Different tasks require different treatment. A single fire-and-forget task is like sending a letter. A cron job is like a daily alarm. A chained task is like an assembly line.

Important Terms

Cron Job — A time-based job scheduler used for recurring tasks.

Chained Task — A workflow where tasks sequentially trigger other dependent tasks.

Key Takeaways

  • Use recurring tasks to keep your database clean.
  • Break down massive jobs into batch or chained tasks to optimize processing.

Quick Revision

  • One-off: Emails, simple notifications.
  • Recurring: Scheduled cleanup, daily reports.
  • Chained: Sequential dependent steps (e.g., video processing).
  • Batch: Bulk concurrent operations.
