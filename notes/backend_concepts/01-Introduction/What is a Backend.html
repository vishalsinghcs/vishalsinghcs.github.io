What is a Backend?

A traditional backend is a computer listening for requests through an open port over the internet to serve content or accept data.

System Components

  • DNS Server: Resolves human-readable domain names into IP addresses.
  • Firewall: A security barrier that only allows traffic through explicitly permitted ports.
  • Reverse Proxy: A server sitting in front of application servers to manage redirects and SSL.
  • Application Server: The actual runtime executing the business logic.

How it works

  1. Client Request: A browser sends a request to a domain.
  2. DNS Resolution: The DNS server uses A records to map the domain to the server's IP.
  3. Firewall Check: The request hits the server's firewall.
  4. Reverse Proxy: Nginx intercepts the request and forwards it to the app server.
  5. Processing: The app server executes logic and responds.

Mental Model

Think of the backend as an office building. The DNS is the address directory. The firewall is the security guard at the door checking IDs. The reverse proxy is the receptionist directing you. The application server is the specific worker.

Important Terms

  • Reverse Proxy — A server that sits in front of web servers.
  • DNS A Record — A DNS record pointing a domain directly to an IP.

Key Takeaways

  • A backend is a computer listening for requests on an open port.
  • Requests pass through DNS, firewalls, and reverse proxies.

Quick Revision

  • DNS uses A records to resolve domains.
  • Firewalls allow Port 80 and Port 443.
  • Nginx acts as a reverse proxy.
