Path and Query Parameters

Static routes are limiting. Dynamic parameters allow routes to act as templates, granting the ability to target specific resources or pass metadata to the server.

Dynamic Routes & Path Parameters

Unlike static routes, a Dynamic Route contains variable sections. These variables are known as Path Parameters (or Route Parameters), because they are physically embedded within the forward slashes of the URL path.

Consider the endpoint to fetch a single user's data. Hardcoding thousands of static routes like /api/users/1, /api/users/2 is impossible. Instead, we use a dynamic route:

GET /api/users/:id

The :id syntax is an industry-wide convention indicating a variable slot. When a request is made to /api/users/123, the server extracts the string "123" and binds it to the id variable. The server's handler can then use that ID to fetch the correct database record.

Semantic Meaning

REST APIs rely heavily on path parameters because they provide a human-readable, semantic construct. The route /api/users/123 clearly reads as: "Fetch the data belonging to the user resource whose ID is 123."

Query Parameters

In HTTP POST or PUT requests, you can send user-defined data inside the request body. However, GET requests traditionally do not have bodies. How do you pass complex data—like search queries or sorting filters—in a GET request?

While you could technically shove everything into path parameters (e.g., /api/search/my+search+value), doing so breaks semantic meaning and makes routing difficult to maintain. Instead, we use Query Parameters.

GET /api/books?sort=asc&limit=20&page=2

Characteristics of Query Parameters

  • They are appended to the very end of the URL, following a question mark (?).
  • They are formatted as key-value pairs (key=value).
  • Multiple parameters are separated by an ampersand (&).

Common Use Cases

  • Pagination: Splitting large datasets into manageable chunks (e.g., ?page=2&limit=20). The server returns a metadata object containing total items, current page, and the data slice.
  • Filtering: Refining results (e.g., ?category=fiction).
  • Sorting: Ordering the response data (e.g., ?order=desc).
  • Searching: Passing a user's search string (e.g., ?query=harry+potter).

Important Terms

  • Path Parameter: Dynamic variables within the route path itself. Used to uniquely identify a resource. Conventionally denoted with a colon (e.g. :id).
  • Query Parameter: Key-value metadata appended to the end of the URL after a ?. Used to filter, sort, or paginate data.

Mental Model

Think of Path Parameters as the exact mailing address identifying a specific person's house. Think of Query Parameters as instructions given to the mail carrier on how to deliver the mail (e.g., "leave at front door", "fragile").

Quick Revision

  • Dynamic routes use path parameters (e.g., /users/:id) as variable slots.
  • Path parameters identify specific resources semantically.
  • Query parameters follow a ? as key-value pairs.
  • Query parameters are used for metadata like filtering, sorting, or pagination, especially in GET requests where bodies are absent.
