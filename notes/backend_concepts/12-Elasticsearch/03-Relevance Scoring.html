Relevance Scoring

Why do we need it?

Finding documents that match a query is only half the battle. In a database of millions, a single query might match tens of thousands of documents. Relevance scoring determines which of those matching documents are the most useful to the user, allowing the engine to return them in a sorted order.

The BM25 Algorithm

Elasticsearch utilizes an algorithm called BM25 (Best Matching 25) to calculate a relevance score for each document against a given query. You don't need to understand the complex mathematics behind it, but you should understand the primary factors it uses to score documents.

Scoring Factors

  • Term Frequency (TF): How often does the search term appear in the document? If "machine" appears 10 times in a document, it is deemed more relevant than a document where it appears only once.
  • Document Frequency (DF): How common is the search term across all documents? Common words (like "the" or "is") carry less weight. Rare words carry more weight and boost relevance significantly.
  • Document Length: How long is the document? A term appearing once in a short tweet is highly relevant. That same term appearing once in a 500-page book is less relevant.

Field Boosting

Field boosting is a practical feature in Elasticsearch that allows developers to manually prioritize certain fields over others during a search. This is crucial for returning the most intuitive results.

For example, if a user searches for a book:

  • A match in the Title might be given a 3x boost.
  • A match in the Description might be given a 2x boost.
  • A match in the Content receives a standard 1x boost.

Mental Model

Relevance scoring is like a judge in a competition. It looks at how often a keyword shows up, how rare the keyword is overall, and where it was placed (title vs. footnote) to declare the winner.

Important Terms

BM25 Algorithm — The default scoring algorithm used by Elasticsearch to rank documents.

Field Boosting — Artificially inflating the relevance score if a match is found in specific, high-priority fields.

Key Takeaways

  • Relevance is what separates great search engines from basic database queries.
  • You can easily customize relevance in Elasticsearch by adjusting field boosts in your JSON query.

Quick Revision

  • Term Frequency: More occurrences = higher score.
  • Document Frequency: Rarer words = higher score.
  • Document Length: Shorter documents = higher density/score.
  • Field Boosting: Title matches outrank body matches.
