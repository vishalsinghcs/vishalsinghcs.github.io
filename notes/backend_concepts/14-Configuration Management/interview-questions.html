Interview Questions

14 - Configuration Management: Security, Validation, and Scalability.

These questions evaluate your deep understanding of how to manage distributed configurations, secure sensitive secrets, debug environment drift, and design fault-tolerant startup sequences.

1. Scenario-based Questions

  • A new backend deployment is failing on startup with a "Cannot connect to database" error in the production environment, but it runs perfectly on staging. The database credentials are mathematically identical. How would you investigate the source of the configuration mismatch?
  • You have a feature flag to enable a new payment gateway, and it is explicitly set to `false` in production configuration. However, a small subset of users report they are seeing the new gateway and it's failing. How would you track down the configuration drift causing this?
  • During a high-traffic event (e.g., Black Friday), your application suddenly crashes with "too many connections" errors from the database. Which configuration parameters would you investigate, and how would you safely tune them in a live system?
  • A junior developer accidentally commits a `.env` file containing production database credentials to a public GitHub repository. What is your immediate incident response plan, and how do you prevent this systemically in the future?
  • Your application uses AWS Parameter Store to fetch configurations at startup. Suddenly, AWS Parameter Store goes down in your region, and your application instances (which are auto-scaling due to high traffic) are failing to boot. How would you architect your configuration loading to survive this upstream outage?
  • You notice that your production logs are filled with massive amounts of noise, including sensitive user payloads, making it impossible to debug real issues. How would you re-architect the application's configuration to handle dynamic log levels safely without requiring a restart?
  • Your backend services need to rotate the JWT signing secret every 30 days for security compliance. However, simply updating the secret instantly invalidates all active user sessions, forcing everyone to log out. How do you manage this configuration rotation gracefully?
  • An application works perfectly on a developer's Mac, but fails instantly in the Dockerized CI/CD pipeline due to a missing environment variable. How do you enforce a strict configuration contract so developers know exactly what variables are required before running the app?

2. "What if?" Questions

  • What if your application blindly trusted all environment variables injected at startup without validating their structure or existence? What are the cascading failure scenarios?
  • What if your team decided to store all configurations (including secrets) in a shared Google Sheet that the application fetches on startup, instead of using a Cloud Secret Manager? Discuss the security and reliability implications.
  • What if you designed a system where the backend application polls a configuration file every 5 seconds to dynamically apply changes, instead of restarting on configuration updates? What are the edge cases and race conditions?
  • What if staging and production shared the exact same Redis cluster, but used different database indexes based on an environment variable, and the variable was accidentally omitted during a deployment?

3. Debugging Questions

  • An Express API route sporadically fails with a 500 error related to a third-party API timeout. The timeout config is clearly set to 5000ms in `config.yaml`, but the logs show the request failing after exactly 1000ms. Where could this override be coming from?
  • Your application is reading from HashiCorp Vault successfully, but certain newly added keys are returning `null` or `undefined` inside the application context, despite being clearly visible in the Vault UI. What layers of the configuration pipeline would you inspect?
  • In a microservices architecture, Service A successfully sends requests to Service B locally. However, in production, Service A throws "connection refused" errors when reaching Service B, even though both are running flawlessly. How do you trace the configuration mismatch?

4. Design Questions

  • Design a robust configuration management system for a monolithic application that needs to pull secrets from AWS Secrets Manager, non-sensitive variables from a `config.yaml` file, and overrides from local environment variables. How do you prioritize them, and when does validation happen?
  • Design a feature flag system from scratch that can enable a feature for exactly 5% of users in the EU region, and allows product managers to flip this flag without needing engineering to redeploy the backend.
  • Design a secure configuration pipeline for an application where frontend developers need access to public API keys (like Stripe Publishable Key) but must strictly not have access to the Stripe Secret Key, while both keys are stored in the same monorepo for infrastructure-as-code.

5. Architecture & Trade-off Questions

  • When managing configurations, what are the trade-offs of baking configuration files directly into a Docker image at build time versus injecting them at runtime via volume mounts or environment variables?
  • When managing secrets in a large distributed system, what are the architectural trade-offs between having each microservice directly fetch its own secrets from a Secret Manager on boot, versus having an orchestration tool (like Kubernetes) fetch them and inject them into the container's environment?
