What is a Backend

A backend is a centralized computer listening on open internet ports (like 80 or 443) to serve content, process logic, or accept data from clients.

Why do we need it?

Backends exist to centralize state, safely persist data, and execute logic that involves multiple users or heavy computation.

  • Data Persistence: Safely saving information to a database.
  • Cross-user Actions: Pushing events from one user to another.
  • Centralized Authority: Acting as the single source of truth for all users.

Real-world Example

Imagine liking a post on a social media app:

  1. You click "Like".
  2. The app sends an HTTP request to the backend.
  3. The backend parses the request, identifies you, and persists the action in the database.
  4. The backend finds the post owner and triggers a push notification to their phone.

Components

  • DNS Server: Resolves human-readable domain names into IP addresses (e.g., using A Records).
  • Firewall: A security barrier that only allows traffic through permitted ports (e.g., 443 for HTTPS).
  • Reverse Proxy: A server (like Nginx) that sits in front of the application to route requests, manage SSL, and handle centralized configs.
  • Application Server: The runtime (like a Node.js server) executing the business logic on a local port.

Mental Model

Think of a backend as an office building. The DNS is the address directory. The Firewall is the security guard checking IDs at the door. The Reverse Proxy is the receptionist routing visitors. The Application Server is the specific worker fulfilling your request.

Important Terms

  • Port — A virtual endpoint where network connections start and end.
  • A Record — A DNS record pointing a domain directly to an IP address.
  • Reverse Proxy — Software that sits between the internet and backend servers to manage routing and traffic.

Key Takeaways

  • Backends are centralized servers acting as the single source of truth.
  • Requests hop through DNS, firewalls, and proxies before reaching business logic.
  • Without backends, applications cannot share state or securely persist data.

Quick Revision

  • Backend = Server runtime processing requests.
  • DNS maps domains to server IPs.
  • Firewalls block unauthorized port traffic.
  • Nginx (Reverse Proxy) routes external traffic to local app ports.
  • Core backend responsibilities: fetch, receive, and persist data.
