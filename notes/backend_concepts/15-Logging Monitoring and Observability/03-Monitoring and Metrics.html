Monitoring and Metrics

Keeping a pulse on your application's health and performance patterns.

Monitoring is the practice of continuously checking the health and performance of your system. Instead of recording individual events (like logging does), monitoring aggregates data to show you patterns and trends over time.

Real-time (with a slight delay)

While monitoring tools aim to give you real-time data, there is usually a slight delay (around 10-15 seconds) in traditional setups. We intentionally introduce this delay to avoid overwhelming the observability infrastructure with constant data floods every single millisecond.

Understanding Metrics

Metrics are the concrete numbers that power monitoring. We use metrics to quantify the state of our system.

  • Infrastructure Metrics: The server's CPU usage, memory utilization (e.g., 3MB garbage collection time), and open database connections in the pool.
  • Performance Metrics: Requests processed per second, average transaction time, and throughput.
  • Business Metrics: How many "to-dos" were created today? How many requests resulted in a failure (status code > 200)?

Alerting based on Trends

Once we have metrics configured, we can set up automated alerts. For example, if a metric indicates that the Error Rate goes above 80%, a webhook can fire an alert directly to your team's Slack channel so an engineer can immediately investigate.

Mental Model

If a log is a single receipt from a store purchase, a metric is the monthly financial report showing revenue trends.

Important Terms

Metrics: Quantifiable measurements of system performance over time.

Throughput: The number of requests your system can successfully process in a given timeframe.

Key Takeaways

  • Monitoring reveals trends, not granular events.
  • A 10-15 second delay is standard to prevent system overload.
  • Metrics drive automated alerts (e.g., Slack notifications).

Quick Revision

  • Infra: CPU, RAM, Network.
  • Performance: Latency, Throughput.
  • Business: Successful transactions, User signups.
