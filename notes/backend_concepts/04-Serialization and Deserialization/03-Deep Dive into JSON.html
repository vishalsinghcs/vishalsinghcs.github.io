Deep Dive into JSON

JSON is the dominant serialization standard for HTTP communication. Let's break down its rules, syntax, and capabilities.

What is JSON?

JSON stands for JavaScript Object Notation. As the name suggests, it was derived from JavaScript, meaning it looks and behaves very similarly to a native JavaScript object.

However, JSON is completely language-agnostic today. You will find it everywhere: in Python data processing, Rust servers, Go microservices, application logs, and configuration files.

The Syntax Rules

While JSON looks like JavaScript, it is much stricter. It relies on a few fundamental rules to ensure any parser (regardless of language) can read it predictably.

  1. Braces: A JSON object must start with an opening curly brace { and end with a closing curly brace }.
  2. Keys: All keys MUST be strings enclosed in exactly double quotes ("). You cannot use single quotes or leave keys unquoted.
  3. Values: Values can only be one of the following fundamental data types:
    • String: Enclosed in double quotes (e.g., "Alice").
    • Number: Integers or floats (e.g., 123 or 3.14).
    • Boolean: true or false.
    • Array: A list of values inside brackets (e.g., [1, 2, 3]).
    • Object: A nested JSON object.
    • Null: Representing an empty value (null).

Example Structure

Here is an example demonstrating all valid JSON data types, including nesting:

{
  "id": 101,
  "name": "John Doe",
  "isActive": true,
  "roles": ["admin", "editor"],
  "address": {
    "country": "India",
    "postalCode": 110001
  }
}

The Lifecycle of JSON

When a frontend client sends this data, it converts its native state into a raw string representing this JSON structure. This raw string is what travels across the wire.

When the backend server receives this raw string, its router passes it to a JSON parser. The parser reads the string, validates the double quotes and braces, and converts it into the backend's native data structure (like a Python Dictionary or Java Class instance) to execute business logic.

Mental Model

Think of JSON as a very strict clipboard. You can copy a complex data structure from one app, put it on this strict clipboard, and paste it perfectly into a completely different app without losing the structure.

Important Terms

  • JSON: A text-based, language-agnostic data interchange format derived from JavaScript.
  • Trailing Comma: A comma placed after the last element in an array or object. Strictly forbidden in JSON.
  • JSON Parser: The engine built into backend languages (like Python's json module) that validates and converts JSON strings into native code objects.

Key Takeaways

  • JSON enforces double quotes for all keys. Single quotes will break the parser.
  • JSON strictly forbids trailing commas, unlike native JavaScript.
  • JSON only supports fundamental types: strings, numbers, booleans, arrays, objects, and null.

Quick Revision

  • JSON stands for JavaScript Object Notation.
  • It is used universally across Python, Rust, Java, etc.
  • Requires {} braces.
  • Keys MUST be double-quoted.
  • Network transmission sends JSON as a raw string.
  • Backend parsers read this string to recreate objects in RAM.
